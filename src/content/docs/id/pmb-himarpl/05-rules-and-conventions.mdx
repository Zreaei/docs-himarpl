Untuk menegakkan gaya kode yang konsisten dan menghindari masalah umum dalam basis kode, kami memiliki serangkaian aturan dan konvensi yang kami ikuti dan tegakkan melalui proyek.

## Typescript

Starter ini menggunakan TypeScript untuk memberikan type safety dan menghindari bug umum dalam basis kode. Konfigurasi proyek didasarkan pada T3 Stack dengan beberapa pembaruan.

Jika Anda tidak familiar dengan Typescript, Anda dapat memeriksa artikel ini untuk mempelajari lebih lanjut tentangnya: [Typescript for React Developers](https://elazizi.com/posts/how-to-learn-typescript-for-react-developers/)

Anda dapat menemukan informasi lebih lanjut tentangnya [di sini](https://create.t3.gg/en/usage/typescript).

## Penamaan

Kami mengikuti kabab-case untuk penamaan file dan folder karena kami pikir itu adalah cara paling mudah dibaca dan konsisten untuk menamai file dan folder dalam proyek besar dan itu adalah cara paling umum untuk menamai file dan folder di komunitas React.

Contoh penamaan kabab-case: `my-component.tsx`

Untuk penamaan variabel, fungsi, kelas, interface, dan enum, kami mengikuti camelCase karena itu adalah cara paling umum untuk menamai variabel di komunitas React.

Contoh penamaan camelCase: `const myFunction = () => {}`

## Linting

Menggunakan linter adalah keharusan dalam proyek JavaScript mana pun. Untuk pemula, kami menggunakan ESLint dengan konfigurasi T3 Stack dan beberapa aturan khusus untuk memastikan bahwa kami mengikuti aturan dan konvensi terkait penamaan file, kelas Tailwind CSS, tipe TypeScript, urutan impor, dan lainnya.

Berikut adalah file konfigurasi ESLint lengkap:
