HIMARPL menggunakan dua database untuk tujuan yang berbeda:

1. **CockroachDB** dengan Prisma ORM - Untuk proyek inti dan infrastruktur utama
2. **PostgreSQL** (NeonDB) dengan Drizzle ORM - Untuk fungsionalitas bot

#### 1. Manajemen Pengguna

- **Pengguna**

  - Informasi profil pengguna inti (nama, email, bio, posisi)
  - Dapat tergabung dalam satu departemen
  - Memiliki akses berbasis peran (admin/anggota)
  - Melacak periode keterlibatan dan aktivitas login

- **Autentikasi**
  - **Akun**: Menangani koneksi OAuth (contoh: login Google)
  - **Sesi**: Mengelola sesi pengguna aktif
  - **Token Verifikasi**: Untuk verifikasi email

#### 2. Manajemen Konten

- **Postingan**

  - Artikel blog dan entri konten
  - Berisi metadata (judul, slug, gambar)
  - Melacak status dan tanggal publikasi
  - Setiap postingan memiliki satu penulis (Pengguna)

- **Tag Postingan**
  - Kategorisasi konten hierarkis
  - Dapat memiliki hubungan induk-anak
  - Digunakan untuk mengorganisir postingan berdasarkan topik

#### 3. Struktur Organisasi

- **Departemen**

  - Dua tipe utama: BE (Badan Eksekutif) dan DP (Dewan Perwakilan)
  - Berisi info departemen (nama, singkatan, deskripsi)
  - Melacak program terkait
  - Pengguna dapat ditugaskan ke departemen

- **Media Sosial**
  - Menghubungkan pengguna ke profil media sosial mereka
  - Menyimpan nama platform, username, dan URL

### Hubungan Kunci

1. **Pengguna & Konten**

- Setiap postingan harus memiliki satu penulis
- Pengguna dapat membuat beberapa postingan
- Pengguna dapat memiliki beberapa profil media sosial

2. **Organisasi Konten**

- Postingan dapat memiliki beberapa tag
- Tag dapat memiliki hubungan induk-anak
- Postingan memiliki slug unik untuk setiap penulis

3. **Organisasi**

- Pengguna dapat tergabung dalam satu departemen
- Departemen dapat memiliki beberapa pengguna
- Setiap pengguna memiliki peran spesifik (admin/anggota)

### Prinsip Desain Skema Prisma

- Type-safe dengan enum yang terdefinisi
- Batasan foreign key yang tepat
- Dapat dengan mudah dikembangkan di masa depan
- Hubungan hierarkis yang jelas

### Aturan Skema Prisma

- Gunakan PascalCase untuk semua models
- Gunakan camelCase untuk semua attributes

## Struktur Database Bot

Bot menggunakan PostgreSQL (NeonDB) dengan Drizzle ORM untuk mengelola notifikasi dan interaksi pengguna.

### Kelas Utama

:::note[Skema Drizzle]
Jika Anda ingin mengintegrasikan proyek baru Anda dengan database, Anda dapat menggunakan Skema Drizzle di bawah ini.

<details>
<summary>Lihat Skema Drizzle</summary>

```typescript {13-31}
import { sql } from "drizzle-orm";
import {
  bigserial,
  index,
  pgTableCreator,
  text,
  timestamp,
  varchar,
} from "drizzle-orm/pg-core";

export const createTable = pgTableCreator((name) => `bot-himarpl_${name}`);

export const notifications = createTable(
  "notification",
  {
    chatId: bigserial("chat_id", { mode: "number" }).primaryKey(),
    firstName: varchar("first_name", { length: 256 }),
    lastName: varchar("last_name", { length: 256 }),
    username: varchar("username", { length: 256 }),
    createdAt: timestamp("created_at", { withTimezone: true })
      .default(sql`CURRENT_TIMESTAMP`)
      .notNull(),
    notifying: text("notifying")
      .array()
      .notNull()
      .default(sql`ARRAY[]::text[]`),
  },
  (notifications) => ({
    chatIdIndex: index("chat_id_idx").on(notifications.chatId),
  })
);
```

</details>

:::

**Tabel Utama: notifications**

- **Tujuan**: Melacak notifikasi chat Telegram dan preferensi pengguna
- **Prefix Tabel**: `bot-himarpl_` (menggunakan pola multi-project schema)

**Field Kunci:**

- `chatId`: Pengenal unik untuk chat Telegram (primary key)
- `firstName`: Nama depan pengguna dari Telegram
- `lastName`: Nama belakang pengguna dari Telegram
- `username`: Username Telegram
- `createdAt`: Timestamp pembuatan chat
- `notifying`: Array topik notifikasi yang diikuti pengguna

**Detail Teknis:**

- Menggunakan tipe PostgreSQL spesifik (bigserial, varchar, timestamp)
- Termasuk indexing pada chatId untuk pencarian lebih cepat
- Mengimplementasikan timestamps dengan timezone
- Menggunakan tipe array untuk preferensi notifikasi

### Prinsip Desain Skema Drizzle

- Menggunakan prefix tabel untuk isolasi multi-project
- Mengimplementasikan tipe PostgreSQL yang tepat
- Termasuk optimasi performa melalui indexing
- Mengikuti praktik terbaik Drizzle ORM

### Aturan Skema Drizzle

- Gunakan camelCase untuk semua models dalam schema
- Gunakan camelCase untuk semua attributes dalam models
- Gunakan snake_case untuk semua nama tabel dalam database
- Gunakan snake_case untuk semua nama attribute dalam database
