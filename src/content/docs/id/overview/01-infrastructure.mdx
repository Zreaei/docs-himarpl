### Arsitektur Tiga Lapis

Setiap proyek mengimplementasikan arsitektur tiga lapis:

#### 1. Lapisan Frontend ([React](https://react.dev/))

UI modern dibangun dengan:

- üé® [TailwindCSS](https://tailwindcss.com/) - Styling berbasis utilitas
- üß© [Shadcn/UI](https://ui.shadcn.com/docs) - Komponen yang dapat digunakan kembali

- üîÑ [Tanstack Query](https://tanstack.com/query) - Pengambilan data

#### 2. Lapisan Tengah ([T3 Stack](https://create.t3.gg/))

Integrasi backend dengan type-safe menggunakan:

- [TypeScript](https://www.typescriptlang.org/) - Keamanan tipe
- [Next.js](https://nextjs.org/) - Framework React
- [tRPC](https://trpc.io/) - API typesafe end-to-end
- [Prisma](https://www.prisma.io/) - ORM Database
- [NextAuth](https://next-auth.js.org/) - Autentikasi

#### 3. Lapisan Backend (Next.js API Routes)

Logika sisi server dengan:

- Proxy analitik ([Integrasi PostHog](https://posthog.com/docs/advanced/proxy/nextjs))
- [tRPC Router](https://trpc.io/docs/server/routers) - Endpoint API
- [tRPC Procedures](https://trpc.io/docs/server/procedures) - Logika bisnis
- [@tRPC Next](https://trpc.io/docs/nextjs) - Integrasi Next.js

## Deployment

:::note
**Catatan Penting untuk Staff Kominfo**

Proyek Vercel baru memerlukan pengaturan GitHub Actions manual:

1. Buat repo GitHub di [HIMARPL UPI](https://github.com/himarplupi)
2. Buat proyek Vercel (gunakan akun GitHub HIMARPL utama, bukan organisasi)
3. Dapatkan token deployment Vercel
4. Siapkan GitHub Actions

[Lihat panduan pengaturan](https://vercel.com/guides/how-can-i-use-github-actions-with-vercel)
:::
