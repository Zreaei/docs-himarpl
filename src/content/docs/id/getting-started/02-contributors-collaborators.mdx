Memahami peran kontributor dan kolaborator dalam proyek HIMARPL

## Apa itu Kontributor?

Kontributor adalah siapa saja yang berkontribusi pada proyek HIMARPL melalui:

- Membuat isu/PR
- Mengomentari isu/PR
- Perbaikan bug
- Penambahan fitur
- Perbaikan dokumentasi
- Pelaporan masalah
- Bentuk kontribusi lainnya

## Apa itu Kolaborator?

Kolaborator adalah anggota tim yang aktif berpartisipasi dalam diskusi proyek dan pengambilan keputusan dengan akses untuk:

- Mereview isu/PR
- Mengelola isu/PR
- Melakukan review kode
- Menerima atau menolak pull request
- Memelihara repositori

Ini termasuk staff HIMARPL dan pengelola web di Departemen Kominfo.

## Memulai Kontribusi

1. Fork repositori
2. Clone fork Anda:
   ```bash
   git clone https://github.com/<NAMA_PENGGUNA_GITHUB_ANDA>/<NAMA_REPO>.git
   ```
3. Siapkan lingkungan pengembangan:
   ```bash
   pnpm install
   pnpm run dev
   ```
4. Buat perubahan Anda
5. Ikuti pedoman commit kami
6. Ajukan pull request

## Pedoman Kontribusi

- Ikuti [Conventional Commits](https://www.conventionalcommits.org/)
- Tandatangani commit Anda (direkomendasikan)
- Ikuti pedoman PR
- Pastikan semua pemeriksaan lulus sebelum penggabungan
- Selesaikan semua diskusi
