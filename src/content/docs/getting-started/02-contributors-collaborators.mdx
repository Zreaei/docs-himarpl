Understanding the roles of contributors and collaborators in HIMARPL projects

## What is a Contributor?

A contributor is anyone who contributes to HIMARPL projects through:

- Creating issues/PRs
- Commenting on issues/PRs
- Bug fixes
- Feature additions
- Documentation improvements
- Issue reporting
- Other forms of contribution

## What is a Collaborator?

A collaborator is a team member who actively participates in project discussions and decision-making with access to:

- Review issues/PRs
- Manage issues/PRs
- Conduct code reviews
- Accept or reject pull requests
- Maintain repositories

This includes HIMARPL staff and web maintainers in the Kominfo Department.

## Getting Started with Contributing

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/<YOUR_GITHUB_USERNAME>/<REPO_NAME>.git
   ```
3. Set up the development environment:
   ```bash
   pnpm install
   pnpm run dev
   ```
4. Make your changes
5. Follow our commit guidelines
6. Submit a pull request

## Contribution Guidelines

- Follow [Conventional Commits](https://www.conventionalcommits.org/)
- Sign your commits (recommended)
- Follow the PR guidelines
- Ensure all checks pass before merging
- Resolve all discussions
